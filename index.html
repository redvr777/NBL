Leaderboard & Data Usage Guide

How the Leaderboard Works

The leaderboard displays all players in the room with their profile pictures and names. Each player has a “Spot on the Leaderboard” showing their position, which updates automatically.

Understanding Spots on the Leaderboard

  • Spot 1 → First player
  • Spot 2 → Second player

Spots update in real-time whenever players join or leave.

Example Leaderboard

  • Spot 1: PlayerOne
  • Spot 2: PlayerTwo
  • Spot 3: PlayerThree

Reporting Players

  • Click “Report” next to a player’s name.
  • Select a reason.
  • Report is submitted and the button hides for that player.

Checking Your Data in the Game

Understand what the leaderboard and reporting system use about you.

Data Used for the Leaderboard

  • Display Name: Shown to all players.
  • Profile Picture: Avatar used in leaderboard and reports.
  • User ID: Unique identifier (not visible to other players).

Data Sent When Reporting

  • Your name and profile picture
  • Reported player’s name and profile picture
  • Selected reason
  • Room name where report occurred

How to Check Your Own Data

  • Open leaderboard in-game to see your spot.
  • Click “Report” to view the data included (in webhook payload, not public).
  • Profile picture and name come from your Meta account.

Tips

  • Update your display name and profile picture in Meta to change in-game appearance.
  • Reports do not expose passwords or sensitive info.
  • Leaderboard spots reflect real-time positions.

Meta Permissions Setup

Required permissions for leaderboard and reporting:

User ID – Active

Access to user ID for multiplayer features.

Why: Required for multiplayer, syncing player data, account management, and content ownership.

User Profile – Active

Access to username and profile photo.

Why: Needed to display avatars, improve identity consistency, and help players recognize each other.

User Age Group – Active

Access to user age group (for mixed-age apps).

Why: Ensures age-appropriate experience, compliance, limits social features if needed, and protects younger users.
